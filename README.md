# ЗАРЯД · Табель учёта рабочего времени

Веб-приложение для учёта рабочего времени на Python. HTTP-сервер на stdlib, база данных SQLite, одна сторонняя зависимость.

## Требования

- Python 3.10+
- pip

## Установка

```bash
pip install -r requirements.txt
```

## Первый запуск

1. Настрой `.env` (скопируй из `.env.example` или создай вручную):

```env
WEB_PORT=9090
WEB_HOST=0.0.0.0
WORKTIME_DB=data/worktime.db
```

2. Создай первого администратора:

```bash
python manage_admins.py add admin ВашПароль
```

3. Запусти сервер:

```bash
python app.py
```

4. Открой в браузере: `http://localhost:9090`

## Управление пользователями (CLI)

```bash
python manage_admins.py list                        # список всех пользователей
python manage_admins.py add <логин> <пароль>        # добавить администратора
python manage_admins.py add <логин> <пароль> accountant  # добавить бухгалтера
python manage_admins.py passwd <логин> <новый_пароль>    # сменить пароль
python manage_admins.py delete <логин>              # удалить
```

Управление пользователями также доступно через веб-интерфейс на странице `/users`.

## Роли

| Роль | Доступ |
|------|--------|
| `admin` | Полный доступ: дашборд, работники, объекты, пользователи, редактирование смен |
| `accountant` | Только просмотр дашборда и скачивание Excel |
| `worker` | Личный кабинет `/my` — своя история смен и heatmap |

## Возможности

- Дашборд с графиком по дням и рейтингом работников по часам
- Массовая отметка прихода / ухода
- Добавление смен задним числом
- Фильтры: авто / открытые / опоздание / с комментарием / по объекту
- Комментарии к сменам (только для admin)
- Профили работников с тепловой картой активности
- Объекты и прикрепление работников к ним
- Заметки к объектам
- Личный кабинет работника (`/my`)
- Экспорт в Excel (.xlsx) с учётом активного фильтра поиска
- Страница управления пользователями (`/users`): добавление, смена пароля, смена роли, удаление
- Пасхальное яйцо 🐷

## Структура проекта

```
app.py              — точка входа, запуск сервера
config.py           — конфигурация и загрузка .env
utils.py            — утилиты (время, периоды, расчёт часов)
sessions.py         — сессии в памяти
manage_admins.py    — CLI управления пользователями
requirements.txt    — зависимости

db/
  conn.py           — подключение и миграции SQLite
  workers.py        — работники
  shifts.py         — смены
  objects.py        — объекты
  attachments.py    — привязка работников к объектам
  credentials.py    — учётные данные работников
  admin_users.py    — администраторы и бухгалтеры
  comments.py       — комментарии к сменам, работникам, объектам
  audit.py          — лог действий

views/
  dashboard.py      — главная страница
  workers.py        — список работников
  worker_profile.py — профиль работника
  objects.py        — список объектов
  object_detail.py  — страница объекта
  users_page.py     — управление пользователями
  my_page.py        — личный кабинет работника
  export.py         — экспорт CSV и Excel
  login.py          — страница входа
  common.py         — общие компоненты (topbar, heatmap)

handler/
  base.py           — базовый HTTP-обработчик
  get_routes.py     — GET-маршруты
  post_routes.py    — POST API

static/
  style.css         — стили
```

## База данных

SQLite, файл задаётся через `WORKTIME_DB` в `.env`. По умолчанию `worktime.db` рядом с `app.py`.  
Миграции применяются автоматически при каждом старте.
